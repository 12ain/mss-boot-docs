## 背景

- 记录时间：2026-06-09
- 执行范围：国内开发者社区内容发布与开源项目回流。
- 执行原则：不刷屏、不求 star、不夸大评分；内容必须围绕开源治理、CI、PR、文档和 AI Agent 协作；外部平台反馈统一回流 GitHub Discussions / Issues / docs。

## 本轮内容主题

- 标题：把开源项目质量交给 GitHub 和 AI：mss-boot-io 的一轮 PR 治理记录
- 定位：面向国内开发者社区的开源治理复盘，不是单纯宣传 CRUD 后台。
- 核心信息：
  - PR 先过 GitHub Actions、CodeQL、govulncheck、PR Guard、OpenSSF Scorecard，再合并。
  - AI Agent 记忆必须落盘到 `mss-boot-docs/aigc`，避免多 Git workspace 中上下文丢失。
  - 依赖安全升级不能盲目硬升，主链路迁移走 RFC。
  - 后端 beta 可以高频验证，前端 Cloudflare beta/prod 需要低频、稳定、冒烟通过。
  - 邀请社区参与 review、issue 复现、测试拆分、RFC 讨论和文档改进。

## 发布后回流动作

- 把每个平台的公开 URL 追加回本文件。
- 到 GitHub Discussions 补一条中文渠道发布汇总评论，继续把有效反馈引导到 GitHub。
- 24-48 小时内巡检评论区：
  - bug / 复现问题：转 GitHub Issues。
  - 架构 / 路线图建议：转 GitHub Discussions。
  - 文档疑问：转 `mss-boot-docs` backlog 或 FAQ。
  - 账号、验证码、平台审核问题：只记录状态，不绕规则。
